Microservices-Based Point of Sale (PoS) System

Explore the modular architecture, features, and technologies powering this scalable PoS solution.

Overview

This project leverages a microservices architecture for a Point of Sale (PoS) system, integrating RabbitMQ for asynchronous messaging, Ocelot as an API Gateway, and Docker for containerization. The system is designed for high scalability and modularity, making it suitable for real-world business needs.

Key Features

Scalability

Services can be independently scaled to handle traffic spikes or specific business needs.

Event-Driven

RabbitMQ ensures decoupled, event-driven communication between services.

API Gateway

Ocelot API Gateway simplifies routing, security, and load balancing for clients.

Architecture

The system follows a modular microservices approach with autonomous services. Each service handles specific business functionalities and communicates via RabbitMQ (asynchronous) or HTTP (synchronous).

  • Customer Service: Manages customer profiles, shopping carts, rewards, and returns.
  • Inventory Service: Handles stock levels, low-stock alerts, and product availability.
  • Sales Service: Manages sales transactions, ticket issuance, and customer rewards updates.
  • Order Management Service: Oversees vendor orders, restocking processes, and order lifecycle management.
  • Employee Service: Manages employee data, roles, and store assignments.
  • Vendor Service: Stores vendor information and handles automated restocking notifications.
  • Tax Service: Calculates and updates tax rates based on customer location and transaction requirements.

Technologies & Tools

ASP.NET Core

RabbitMQ

Ocelot
